1. Objetivo

Registrar automáticamente quién modifica las tablas espaciales, cuándo y qué registro fue afectado. Permite conocer la frecuencia de actualización de cada capa por parte de las distintas áreas municipales.

El sistema utiliza triggers de PostgreSQL que se disparan automáticamente ante cualquier INSERT, UPDATE o DELETE, sin requerir intervención manual.

Script de creación

CREATE TABLE IF NOT EXISTS sc_catastro.auditoria_tablas (
    id            SERIAL PRIMARY KEY,
    schema_name   TEXT NOT NULL,
    table_name    TEXT NOT NULL,
    user_name     TEXT DEFAULT current_user,
    action_time   TIMESTAMP WITH TIME ZONE DEFAULT clock_timestamp(),
    action        TEXT NOT NULL,  -- 'INSERT', 'UPDATE' o 'DELETE'
    original_data TEXT,
    new_data      TEXT,
    registro_id   BIGINT
);

4. Función trigger universal

La función es universal: un solo objeto maneja la auditoría de todas las tablas. Las variables TG_TABLE_NAME y TG_TABLE_SCHEMA capturan automáticamente el contexto de ejecución.

CREATE OR REPLACE FUNCTION sc_catastro.fn_auditar_tabla()
RETURNS TRIGGER AS $$
BEGIN
    IF TG_OP = 'DELETE' THEN
        INSERT INTO sc_catastro.auditoria_tablas
            (schema_name, table_name, user_name, action_time, action, registro_id)
        VALUES
            (TG_TABLE_SCHEMA, TG_TABLE_NAME, current_user, NOW(), TG_OP, OLD.id);
    ELSE
        INSERT INTO sc_catastro.auditoria_tablas
            (schema_name, table_name, user_name, action_time, action, registro_id)
        VALUES
            (TG_TABLE_SCHEMA, TG_TABLE_NAME, current_user, NOW(), TG_OP, NEW.id);
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
Nota: OLD.id se usa para DELETE porque el registro ya no existe. NEW.id se usa para INSERT y UPDATE.

5. Creación de triggers

Trigger — Luminarias servicios

DROP TRIGGER IF EXISTS trg_luminarias_auditoria ON servicios.luminarias_26052026;
CREATE TRIGGER trg_luminarias_auditoria
AFTER INSERT OR UPDATE OR DELETE
ON servicios.luminarias_26052026
FOR EACH ROW EXECUTE FUNCTION sc_catastro.fn_auditar_tabla();

Trigger — Loteos sc_catastro

DROP TRIGGER IF EXISTS trg_loteos_auditoria ON sc_catastro.loteos_definitivos_11032024;
CREATE TRIGGER trg_loteos_auditoria
AFTER INSERT OR UPDATE OR DELETE
ON sc_catastro.loteos_definitivos_11032024
FOR EACH ROW EXECUTE FUNCTION sc_catastro.fn_auditar_tabla();

Trigger — Red Vial sc_base

DROP TRIGGER IF EXISTS trg_red_vial_auditoria ON sc_base."RED_VIAL";
CREATE TRIGGER trg_red_vial_auditoria
AFTER INSERT OR UPDATE OR DELETE
ON sc_base."RED_VIAL"
FOR EACH ROW EXECUTE FUNCTION sc_catastro.fn_auditar_tabla();
Importante: Las tablas con nombres en MAYÚSCULAS (como RED_VIAL) requieren comillas dobles. Las tablas en esquemas distintos a public requieren el prefijo esquema.tabla.
Tip DBeaver: Si se produce un error de transacción, ejecutar ROLLBACK; antes de volver a correr los scripts.

6. Permisos necesarios

Si el rol que edita desde QGIS no puede insertar en la tabla de auditoría, otorgar permisos:

-- Reemplazar 'catastro' con el nombre del rol correspondiente
GRANT INSERT ON sc_catastro.auditoria_tablas TO catastro;
GRANT USAGE, SELECT ON SEQUENCE sc_catastro.auditoria_tablas_id_seq TO catastro;

7. Agregar una nueva tabla al sistema

Para auditar una nueva tabla, solo se necesita crear el trigger. No hace falta crear una nueva función ni una nueva tabla de auditoría.

DROP TRIGGER IF EXISTS trg_NOMBRETABLA_auditoria ON esquema.NOMBRETABLA;
CREATE TRIGGER trg_NOMBRETABLA_auditoria
AFTER INSERT OR UPDATE OR DELETE
ON esquema.NOMBRETABLA
FOR EACH ROW EXECUTE FUNCTION sc_catastro.fn_auditar_tabla();
Nota: Si la nueva tabla tiene su columna ID con un nombre distinto a id (por ejemplo gid o objectid), actualizar la función para que use el nombre correcto.

8. Verificaciones

Confirmar que los triggers apuntan al esquema correcto

SELECT t.tgname AS trigger_name,
       p.pronamespace::regnamespace AS funcion_esquema,
       p.proname AS funcion_nombre
FROM pg_trigger t
JOIN pg_proc p ON t.tgfoid = p.oid
JOIN pg_class c ON t.tgrelid = c.oid
WHERE t.tgname IN (
    'trg_luminarias_auditoria',
    'trg_loteos_auditoria',
    'trg_red_vial_auditoria'
);

9. Consultas de uso cotidiano

Última actividad por tabla

SELECT table_name, action, user_name, MAX(action_time) AS ultima_vez
FROM sc_catastro.auditoria_tablas
GROUP BY table_name, action, user_name
ORDER BY ultima_vez DESC;

¿Cuándo tocaron una tabla específica?

SELECT MAX(action_time) AS ultima_actualizacion
FROM sc_catastro.auditoria_tablas
WHERE table_name = 'RED_VIAL';

¿Qué IDs se modificaron en loteos?

SELECT registro_id, action, user_name, action_time
FROM sc_catastro.auditoria_tablas
WHERE table_name = 'loteos_definitivos_11032024'
ORDER BY action_time DESC;

Resumen mensual de actividad

SELECT
    table_name,
    DATE_TRUNC('month', action_time) AS mes,
    action,
    COUNT(*) AS cantidad_filas
FROM sc_catastro.auditoria_tablas
GROUP BY table_name, mes, action
ORDER BY mes DESC, table_name;

¿Quién actualizó y cuántas veces?

SELECT
    table_name,
    user_name,
    action,
    COUNT(*)         AS veces,
    MIN(action_time) AS primera_vez,
    MAX(action_time) AS ultima_vez
FROM sc_catastro.auditoria_tablas
GROUP BY table_name, user_name, action
ORDER BY ultima_vez DESC;

10. Notas importantes

Tabla vacía: La tabla auditoria_tablas estará vacía hasta que alguien modifique una de las tablas auditadas — es el comportamiento esperado.
Historial previo: Los registros anteriores a la implementación del sistema no quedan registrados — solo se registran cambios desde la fecha de instalación.
Error de transacción: Si se produce un error en DBeaver, ejecutar ROLLBACK; antes de volver a correr los scripts.
Columna ID: El campo registro_id captura el id de la fila afectada. Si la tabla auditada usa otro nombre de columna como clave (gid, objectid, etc.), actualizar la función acordemente.
